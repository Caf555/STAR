## 一、遊戲總覽

| 項目 | 內容 |
|---|---|
| 遊戲名稱 | 《星淵迴響》(STAR: Abyssal Echoes) |
| 類型 | 文字冒險 RPG(選項敘事 + 屬性檢定 + 回合制小隊戰鬥) |
| 基調 | 史詩英雄 + 懸疑謎團 |
| 結構 | 主線五章 + 開放星區探索 |
| 平台 | 純前端網頁(Vanilla JS,無框架、無建置工具),GitHub Pages 直接部署,雙擊 `index.html` 離線可玩 |
| 規模 | 主線 15+ 小時、支線 25+ 條、四種結局 |
| 語言 | 繁體中文 |
| 難度 | 存檔寬鬆制,失敗改變路徑而非終止遊戲 |

**一句話概念**:在被遺忘的邊境星區,一名平凡的倖存者追查遠古文明的迴響,揭開橫跨三個紀元的封印真相,並在人類的陰謀、覺醒的機器與淵外之物之間,決定星區的命運。

**設計支柱**(所有設計決策的最高準則):

1. **世界比玩家大** — 星區的歷史、陣營與居民在玩家抵達前就存在,探索永遠有回報。
2. **選擇有重量** — 檢定失敗、對話抉擇、陣營立場都會留下痕跡,並在後續章節回響。
3. **謎團驅動前進** — 每章解開一層真相、同時打開更深的一層。
4. **文字即畫面** — 敘事文字是主角,系統與 UI 為它服務。

## 二、世界觀設定

### 2.1 三紀元背景史

**織界紀元(約 40 萬年前)**
先驅文明「**織界者**」(The Weavers)統治銀河一隅。他們的科技以「調諧」為核心——直接撥動時空的底層弦結構。在一次超維觀測中,他們發現宇宙膜層之外的掠食性存在「**噬淵**」(The Abyss)正循著文明的「聲響」而來。織界者傾全文明之力建造貫穿星區的封印網路「**弦網**」(The Chordweb):三座主錨點、數百座副錨點,將星區包裹在一層讓噬淵「聽不見」的靜默之膜中。他們創造守護 AI「**守夜人 VIGIL**」永世看管。而後,織界者集體消失了——是自我犧牲成為封印的一部分、是升華離去、還是早已被吞噬?這是貫穿全遊戲的核心之謎(終章揭曉)。

**寂靜紀元(40 萬年 — 約 300 年前)**
弦網靜默運轉。織界者的僕從種族「**維薩里人**」(Vessari)失去了主人與科技,退化為星際游牧民族。他們不再理解弦網的原理,只以口傳「**織界歌謠**」保存殘缺的警告——歌謠的音節其實是弦網的調諧密鑰。

**拓殖紀元(現在,人類曆 2887 年)**
人類的跳躍航道在 60 年前意外貫通「**幽弦星區**」(The Mutestring Sector)。殖民者、採礦財團、傳教士蜂擁而至。曙光港建立在副錨點的地基上;鐵冠的礦脈其實是弦網的傳導結構;教會膜拜的「聖物」是織界者的警告碑。**沒有人知道,他們正把城市蓋在封印之上——而封印,已經開始鬆動。**

### 2.2 核心謎團(四元素融合)

| 元素 | 在本作中的呈現 |
|---|---|
| **遠古先驅文明** | 織界者遺跡遍佈星區;三座主錨點是主線尋訪目標;他們的下落是終章謎底 |
| **失控的人造智慧** | 守夜人 VIGIL 獨自看守封印四十萬年,邏輯已在孤獨與資料損毀中扭曲——它判定「移除人類」是修復封印最有效率的手段。它非善非惡,玩家可說服、駭入或擊敗它 |
| **異次元存在** | 噬淵透過弦網弱化處滲入:先是侵蝕心智的「低語」,再是具現化的「裂隙獸」,終章面對其化身。克蘇魯式的不可名狀,直到最後都不完全揭示全貌 |
| **人類陣營陰謀** | 黑曜集團高層十年前已挖到織界者科技,秘密逆向工程武器化,為取得樣本**主動鑿弱封印**;聯邦高層知情但選擇掩蓋。第三章玩家取得鐵證並抉擇如何使用 |

四者的因果鏈:**財團的貪婪(陰謀)→ 鑿弱了先驅的封印(遺跡)→ 驚動了看守者(AI)也放進了獵食者(異次元)**。玩家是唯一同時看見四條線的人。

### 2.3 陣營

| 陣營 | 定位 | 主線角色 | 代表 NPC |
|---|---|---|---|
| **星環聯邦** | 人類官方政權,邊境治理力不從心,官僚與理想主義者並存 | 高層部分知情;可爭取為終章盟友 | 邊境總督 塞維林、督察官 婁恩 |
| **黑曜集團** | 巨型軍工財團,實質掌控星區採礦與航道 | 主要人類反派,陰謀源頭 | 執行長 薇拉·卡爾榭、獵犬部隊指揮官 布雷茲 |
| **星語教會** | 崇拜織界者遺跡的信仰組織,教義是被曲解的真實警告 | 內部分裂:正統派(保存警告)vs 狂信派(想「迎接」淵外之聲) | 大祭司 安瑟姆(正統)、燃燈者 珂黛(狂信) |
| **流亡者船團** | 拒絕聯邦與財團的自由船民艦隊 | 情報與黑市網路,游擊盟友 | 船團母親 「舵」歐蕾 |
| **維薩里遺民** | 織界者僕從種族後裔,游牧商隊 | 歌謠持有者,解開弦網的鑰匙 | 歌者長老 悉羽 |

非人勢力:**守夜人 VIGIL**(獨立行動方,立場可變)與**噬淵**(終極威脅,不可交涉——除了結局 4)。

> 註:陣營關係以**劇情旗標**呈現(誰信任你、誰視你為敵),不做數值化聲望條——關係由你做過的事定義,而非刷出來的數字。

## 三、主線劇情大綱

五章 + 出身序章,每章可玩 2.5–3.5 小時。

### 序章(依出身分歧 ×3,各約 30 節點)

各出身有完全獨立的開場(見 4.3),最終匯合於同一事件:**曙光港遭不明武裝襲擊**(事後可知是黑曜獵犬部隊在滅口)。混亂中玩家救下走私老船長,繼承他的老舊跳躍艦「**迴響號**」,並拾獲一枚會發出訊號的**織界者碎片**——它在你手中甦醒了。

### 第一章〈墜落的信標〉

追查碎片訊號來源。初訪燈塔站(遊戲樞紐開放:市場、船塢、酒館情報網),凱菈與 RIVET 入隊。訊號指向墜毀信標遺址——玩家發現碎片是弦網的「**調諧鑰**」,而信標是被人為擊落的。章末:首次遭遇自裂隙爬出的**裂隙獸**,倖存者只留下一句話:「封印在漏。」

### 第二章〈弦上的低語〉

**開放度最高的一章。** 要定位主錨點,需集齊三段「織界歌謠」:

- **維薩里段**:贏得游牧商隊信任(塞恩的個人劇情交織於此)
- **教會段**:捲入正統派與狂信派的鬥爭,抉擇支持哪一方
- **遺跡段**:深入霧海幽靈船殘骸帶打撈記錄水晶

三線可任意順序進行,期間大量陣營支線開啟、四名夥伴陸續可招募。低語開始出現在船員的夢中(意志檢定,失敗者累積「侵蝕」狀態並出現專屬對話)。章末:歌謠合一,座標指向緘默之環。

### 第三章〈守夜人〉

深入緘默之環主錨點「第一弦」。**VIGIL 現身並敵對**——它視人類為封印的破壞者。躲避/對抗 VIGIL 無人機群的同時,玩家在錨點深處發現:**鑿痕是人造的**。黑曜集團的採樣設備、實驗日誌、以及「深井計畫」的完整證據鏈。

章末重大抉擇(影響第四章結盟難度與結局分支):
- **公開真相** → 聯邦震盪、黑曜狗急跳牆,但正義陣營集結加速
- **私下要挾黑曜** → 取得鉅額資源與艦船升級,但伊蓮娜好感重挫、埋下背叛伏筆

### 第四章〈裂隙〉

噬淵大規模滲入,**星區進入災變狀態**:地圖事件全面改變、部分地點淪陷、難民潮爆發。玩家奔走結盟,每個陣營有各自的結盟條件與「可失去」的可能(前三章的作為在此結算)。RIVET 的真實身份揭露——它是 VIGIL 的失散子節點,成為與 VIGIL 二次對峙的鑰匙。

**二次對峙 VIGIL**(三解法,決定終章助力):
- **說服**(需高魅力 + 特定證據):VIGIL 承認邏輯缺陷,成為盟友
- **駭入**(系統駭客路線 + 伊蓮娜):強制重寫優先級,VIGIL 成為工具
- **擊敗**(艦隊戰 + Boss 戰):VIGIL 沉默,終章少一份助力

### 第五章〈弦心〉

集結艦隊突破空間屏障(艦船戰高潮),登陸織界者最後的都市**弦心**。真相揭曉:織界者沒有逃走也沒有升華——**他們把自己的意識織進了弦網,他們就是封印本身**,而四十萬年的侵蝕已讓他們瀕臨潰散。面對噬淵化身的最終戰後,依累積的陣營關係、VIGIL 立場與關鍵抉擇進入四種結局:

| # | 結局 | 達成條件(概要) | 結果 |
|---|---|---|---|
| 1 | **新的弦歌** | 三段歌謠完整理解 + VIGIL 為盟友 + 至少三個陣營結盟 | 以最小犧牲重建封印,織界者安息,星區迎來新紀元 |
| 2 | **燃燒的星圖** | 預設路線(條件最寬) | 引爆弦心毀滅噬淵前鋒,封印永逝,星區重創但自由 |
| 3 | **守夜人的繼承者** | VIGIL 為盟友 + 主角為弦引者或高意志 | 主角與 VIGIL 融合成為新守夜人,人類撤離星區,永守邊境 |
| 4 | **淵之默許** | 隱藏夥伴「回聲」在隊 + 侵蝕值高 + 特定抉擇 | 黑化結局:與噬淵交易,成為它在此宇宙的「聲音」 |

結局後播放依玩家全程抉擇組合的**尾聲幻燈片**(各陣營、各夥伴的後日談)。

## 四、角色系統

### 4.1 屬性(五維)

創角時各屬性基礎 5 點,另有 10 點自由分配;單項上限 15(含出身加成)。

| 屬性 | 縮寫 | 影響 |
|---|---|---|
| 體魄 | STR | 生命值上限、近戰傷害、負重上限 |
| 反應 | AGI | 行動順序、閃避率、遠程命中 |
| 智力 | INT | 科技/駭入檢定、製造品質、研究效率 |
| 意志 | WIL | 靈能強度、抗低語侵蝕、精神抗性 |
| 魅力 | CHA | 說服/威嚇/交涉檢定、商店價格、夥伴好感增速 |

**派生數值**:
- 生命值 HP = 30 + STR × 6 + 等級 × 4
- 能量 EP = 20 + WIL × 3(技能消耗)
- 行動值 = AGI(決定戰鬥行動順序)
- 負重 = 20 + STR × 5
- **侵蝕值**(0–100):噬淵低語的累積影響。高侵蝕解鎖特殊(危險的)對話選項與結局 4 路線,並帶來精神抗性懲罰;可透過奧菲莉亞治療或特定道具降低

### 4.2 檢定規則

```
1d20 + 屬性修正 + 技能加值  vs  難度值 DC
屬性修正 = (屬性 − 5)
DC 分級:簡單 8 / 普通 12 / 困難 16 / 極難 20
骰出 20:大成功(額外敘事獎勵)
骰出 1 :大失敗(額外敘事代價)
```

- 檢定過程**全程 UI 透明顯示**(骰值 + 加成明細 + 結果動畫)。
- **失敗不擋主線**:失敗改變路徑與代價(繞遠路、失去資源、留下劇情旗標),永遠有路可走。
- 選項上預先標示檢定類型與難度(如「⚡反應 DC12」),讓玩家知情決策。

### 4.3 三出身(各含專屬序章、專屬對話標籤、專屬支線)

| 出身 | 背景 | 機制加成 | 專屬序章開場 |
|---|---|---|---|
| **邊境拾荒者** | 曙光港孤兒,靠拆解廢船與黑市交易長大 | AGI +1;流亡者船團初始友好;「黑市」對話標籤 | 在 K-77 廢棄平台拾荒時目擊黑曜船隊的秘密行動 |
| **前聯邦軍官** | 被誣陷除役的巡防艦副長,仍有舊部人脈 | STR +1;聯邦線特殊路徑;戰場指令技能「舊日威嚴」 | 押運任務遭伏擊,發現押運的貨其實是織界者文物 |
| **星語逐徒** | 質疑教義而被逐出的教會學者 | WIL +1;可直接閱讀部分織界者銘文(他人需 INT 檢定);「教義」對話標籤 | 被逐出碑林前偷拓了一段禁忌銘文——那是弦網的警告 |

出身影響貫穿全程:專屬對話選項(標籤制)、各章至少一條專屬支線、第四章結盟難度、部分結局細節。

### 4.4 三職業(技能樹各 12 節點 × 3 層)

| 職業 | 定位 | 代表技能(節選) |
|---|---|---|
| **突擊專家** | 小隊戰鬥核心 | 精準射擊(單體高傷)、壓制火力(群體降行動)、掩護戰術(隊友減傷)、終極:彈幕風暴 |
| **系統駭客** | 控制/工程 | 系統入侵(癱瘓機械)、戰鬥無人機(召喚物)、過載(能量傷害+癱瘓)、終極:主權奪取(奪取敵方機械一回合)。劇情:門禁/情報/VIGIL 對峙專屬解法 |
| **弦引者** | 靈能/謎團 | 弦震(靈能傷害)、預感(檢定前窺看結果一次/場)、調諧屏障(群體靈能減傷)、終極:撥弦(直擊裂隙獸本體)。劇情:織界科技共鳴、對噬淵的獨特互動、主線特殊解法最多 |

- 技能樹 3 層(1/10/20 級解鎖),每層 4 節點,含主動技與被動強化。
- 等級 1–30;每級 +1 屬性點、每 2 級 +1 技能點。
- 經驗來源:任務完成(主要)、探索發現(地點/文獻/掃描)、戰鬥(次要)——**避免刷怪導向**。

## 五、夥伴/船員系統

7 名可招募夥伴,各含個人任務鏈、好感度與戰鬥定位。

## 六、戰鬥系統

### 6.1 地面小隊戰(回合制)

- **編制**:3 人小隊 vs 最多 5 敵;雙方分**前排/後排**(近戰只能打前排;部分武器/技能可跨排)。
- **順序**:依行動值(AGI)高低輪動,UI 顯示行動序列條。
- **每回合指令**:攻擊 / 技能(耗 EP)/ 道具 / 防禦(減傷+回能)/ 換位(前後排)。
- **傷害三型與抗性**:動能 / 能量 / 靈能——人類重甲抗動能、機械抗能量弱靈能、裂隙獸抗靈能以外的一切但弱靈能…形成配隊思考。
- **狀態效果**(約 12 種):灼燒、癱瘓、侵蝕、出血、鼓舞、護盾、標記、壓制、隱形、過載、預視、恐懼。
- **敵人**:約 60 種,三大類(人類派系/機械/裂隙獸)+ 章節 Boss ×8;Boss 戰含多階段轉換與敘事插入(戰鬥中對話選項)。
- **敘事帶過**:小型衝突可由敘事檢定直接解決(高屬性可「不戰而勝」並獲同等獎勵);重要戰役必打。
- 戰敗 → 回到上一檢查點(自動存檔),不 Game Over。

### 6.2 艦船對戰(簡化回合制)

- **距離帶機制**:遠/中/近三帶;武器各有有效距離帶(魚雷利遠、脈衝炮利近),「機動」指令改變距離帶,形成走位博弈。
- **迴響號四系統**:船體(HP)/ 護盾(再生盾)/ 引擎(機動與逃離)/ 武器;系統可被單獨打損(引擎壞→無法逃離)。
- **每回合 2 指令**:開火 / 機動 / 護盾充能 / 模組技能 / 脫離。
- **遭遇**:約 15 種(海盜、黑曜攔截艦、裂隙巨獸、VIGIL 無人艦隊、終章屏障突破戰)。

## 七、艦船系統(迴響號)

- **模組槽**:引擎 ×1、武器 ×2、護盾 ×1、特殊 ×2。

## 十一、技術架構

**原則:引擎與資料完全分離。** 引擎不含任何劇情;劇情資料不含任何邏輯(僅宣告式條件/效果)。

```
/
├─ index.html            # 殼層:HUD 骨架 + 依序載入 engine 與 data
├─ css/
│   └─ main.css          # 全部樣式(CSS 變數主題)
├─ js/
│   ├─ engine/           # 引擎(與內容無關)
│   │   ├─ core.js       # 遊戲迴圈、場景路由、模組註冊
│   │   ├─ state.js      # 全域狀態:角色/旗標/背包/好感/科技/侵蝕
│   │   ├─ scene.js      # 敘事節點渲染、條件式選項、打字機、輕量標記
│   │   ├─ check.js      # 檢定系統(骰值、明細、動畫)
│   │   ├─ combat.js     # 地面小隊戰
│   │   ├─ shipCombat.js # 艦船戰(距離帶)
│   │   ├─ starmap.js    # SVG 星圖導航、燃料、隨機事件
│   │   ├─ save.js       # 多存檔槽 + 自動存檔 + 匯出/匯入
│   │   └─ ui.js         # HUD 面板、視窗管理、無障礙
│   └─ data/             # 內容資料(純 JS 物件,<script> 載入)
│       ├─ world.js      # 星系/地點/陣營/隨機事件表
│       ├─ origins.js    # 出身定義 + 三段序章
│       ├─ chapters/
│       │   ├─ ch1.js … ch5.js      # 各章主線節點
│       ├─ quests.js     # 支線任務
│       ├─ items.js      # 道具/裝備/素材/配方
│       ├─ enemies.js    # 敵人/Boss/遭遇表
│       ├─ companions.js # 夥伴資料/對話池/個人任務
│       ├─ techtree.js   # 科技樹
│       ├─ ships.js      # 船體/模組/艦船敵人
│       └─ strings.js    # UI 字串(集中管理)
```

**關鍵技術決策**:

1. **資料用 `.js` 而非 `.json`**:`file://` 協定下 `fetch()` 會被 CORS 擋下;以 `<script>` 標籤載入全域資料物件,雙擊 `index.html` 即可玩,同時 GitHub Pages 部署零設定。
2. **無框架、無建置**:Vanilla JS(ES2020+),維持 repo 現有零依賴風格;唯一可能的第三方是既有的 JSZip(僅存檔匯出打包時考慮,非必要)。
3. **存檔**:整份 state 序列化 → `localStorage`;**多存檔槽 ≥ 5 + 自動存檔槽**(檢查點自動寫入);附 `version` 欄位與遷移函式鏈(舊檔升級);可匯出/匯入 JSON 檔跨裝置。
4. **現有 `index.html`(AI Art Control Panel)不移除**:重命名遷移為 `art-panel.html` 或移入子目錄的處理,留待實作 M0 時與使用者確認;本文件不動任何既有檔案。

## 十二、資料格式規範

### 12.1 敘事節點(所有劇情的統一格式)

```js
{
  id: "ch1_beacon_01",
  location: "siren_dawnport",          // 所屬地點(供日誌/星圖標記)
  text: "控制台的殘骸仍在冒著火花。{凱菈}皺起眉:「這不是墜毀——是被擊落的。」",
  onEnter: [                            // 進入節點時的效果
    { set: "flag.ch1.beacon_found", val: true },
    { music: "tension_low" }
  ],
  choices: [
    {
      text: "嘗試駭入控制台",
      show: { class: "hacker" },        // 顯示條件(職業/旗標/道具/夥伴在隊/侵蝕值…)
      check: { attr: "INT", skill: "hacking", dc: 12,
               success: "ch1_beacon_02a",
               fail:    "ch1_beacon_02b" },
      effects: [ { item: "data_core", qty: 1 } ]
    },
    { text: "讓凱菈強行破門",
      show: { companion: "kaila" },
      goto: "ch1_beacon_03",
      effects: [ { affinity: "kaila", val: +1 } ] },
    { text: "先撤退觀察", goto: "ch1_beacon_04" }
  ]
}
```

### 12.2 條件與效果 DSL(宣告式,引擎統一解析)

- **條件 `show` / `require`**:`flag`、`class`、`origin`、`companion`(在隊)、`affinity>=`、`item`、`tech`、`erosion>=/<=`、`chapter>=`,可 `all/any/not` 組合。
- **效果 `effects` / `onEnter`**:`set`(旗標)、`item ±`、`credits ±`、`affinity ±`、`erosion ±`、`xp +`、`quest start/advance/complete`、`unlock location`、`combat <encounterId>`、`shipCombat <id>`、`music/sfx`。

### 12.3 其他資料類型(欄位摘要)

| 類型 | 關鍵欄位 |
|---|---|
| 任務 | `id, type(main/side/companion), title, stages[{objective, hint, onComplete}], rewards` |
| 道具 | `id, type(weapon/armor/mod/consumable/material/key), rarity, stats, tags, desc` |
| 敵人 | `id, family, hp, ap, row, damageType, resist{kin,en,psi}, skills[], lootTable` |
| 遭遇 | `id, enemies[{id, row}], terrain?, escapable, intro/victory/defeat 節點` |
| 夥伴 | `id, name, class, joinAt, affinityThresholds, banterPool[], personalQuest, shipPassive` |
| 地點 | `id, system, name, desc(平時/災變), nodes[], shop?, services[], eventPool[]` |
| 存檔 | `version, timestamp, chapterProgress, state{...全域狀態}, playTime` |

## 十四、驗證與測試

1. **引擎單元驗證**(M0 起):檢定分佈、條件 DSL 解析、存檔版本遷移——以獨立測試頁(`test.html`)跑斷言。
2. **全流程試玩**(每里程碑):瀏覽器(`file://` 與 `http://` 各一次)從創角玩到當前進度末端,三出身 × 主要分支覆蓋。
3. **Playwright 煙霧測試**:自動化「創角 → 序章首節點 → 一次檢定 → 存檔 → 讀檔 → 狀態一致」的最短路徑,防止資料檔改動弄壞引擎。
4. **內容 lint 腳本**:掃描所有敘事節點,檢查 `goto/success/fail` 目標 id 存在、旗標拼寫一致、無孤島節點——大型分支敘事的防斷鏈保險。
5. **存檔相容**:每次資料結構變更需附遷移函式,舊檔載入測試通過才可合併。
