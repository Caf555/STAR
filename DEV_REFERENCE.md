## 三、可複用的架構模式

### 3.1 引擎與資料徹底分離
引擎不含任何劇情;劇情不含任何程式邏輯,只有**宣告式的條件/效果 DSL**:

```js
{ id: "ch1_beacon_01",
  text: "…{凱菈}皺起眉…",                       // 輕量標記:{人名} 上色、**重點**高亮
  onEnter: [{ set: "flag.x" }, { item: "data_core", qty: 1 }],
  choices: [{
    text: "嘗試駭入控制台",
    show: { class: "hacker" },                    // 條件 DSL:all/any/not/flag/item/quest…
    check: { attr: "INT", skill: "hacking", dc: 12,
             success: "ch1_02a", fail: "ch1_02b" },
    effects: [{ credits: -20 }]
  }] }
```

**為什麼值得**:五個章節、序章、支線、個人任務全部長一個樣,引擎一次寫對就不必再動;而且宣告式資料可以被機器檢查——見 5.2 內容 lint,這在分支敘事規模化之後是救命的。

### 3.2 `file://` 相容:資料用 `.js` 不用 `.json`
`fetch("data.json")` 在 `file://` 協定下會被 CORS 擋死。改為每個資料檔是一個 IIFE,往全域 `SE.DATA` 上 `Object.assign`,以 `<script>` 標籤載入——雙擊 index.html 即可玩,GitHub Pages 也零設定。

### 3.3 存檔:整棵狀態樹序列化 + 版本遷移鏈
- 單一狀態樹 `State.data`(角色/旗標/背包/任務/艦船/科技),存檔 = `JSON.stringify` 全樹。
- 存檔物件帶 `version`;`migrations[n]` 把版本 n 升到 n+1,讀檔時**鏈式升級到最新**。每次狀態結構加欄位(艦船、科技樹…)都補一個遷移函式,舊玩家的存檔永不作廢。
- 匯出/匯入 JSON 檔 = 零後端的跨裝置方案。

### 3.4 場景路由的單一入口
所有節點跳轉都走 `Core.goto(nodeId, opts)` → `Scene.show()`。存檔點、HUD 刷新、氛圍音樂切換、打字機都掛在這一個入口上——後來要加任何「每個節點都要做的事」(如 `updateMood()`)只改一處。

### 3.5 Web Audio:生成式配樂而非「音檔」或「持續音」
- **音訊圖**:`master ← musicBus / sfxBus`,兩條匯流排各自音量;共用一顆用「指數衰減雜訊」現做脈衝響應的 ConvolverNode 當殘響——空間感是廉價但回報最大的一步。
- **音樂**:前瞻排程器(`setInterval` 90ms,排未來 0.35s 的音符)驅動;每情境一份樂譜規格(調性、BPM、和弦進行、配器 pattern),旋律逐小節以「和弦內音為錨的隨機漫步」生成——**永不重複、零音檔**。
- **SFX**:每個音效 = 衝擊層(低頻快攻擊)+ 質感層(特徵中頻)+ 尾音層,再乘 ±3% 隨機音高,連續觸發不機械。
- **手勢限制**:瀏覽器要求使用者手勢後才可出聲。用一個 document 級 click 委派監聽統一呼叫 `ctx.resume()`,順便播 UI 音——一石二鳥。

### 3.6 免建置的測試(見第五節)
`test.html` 一個頁面 = 單元測試 + 內容 lint;Playwright 腳本 = 端對端全流程。都不需要 npm build。

## 四、關鍵事項提醒:踩坑實錄

以下每條格式:**症狀 → 根因 → 對策**。全部真實發生。

### 4.1 資料載入順序會靜默覆蓋定義
- **症狀**:戰鬥中「使用道具」按鈕無故消失。
- **根因**:示範場景資料檔較晚載入,用 `Object.assign` 重複定義了 `stim_patch`,把 items.js 版本的 `combat: { heal: 20 }` 欄位**整個蓋掉**。JS 不會警告你。
- **對策**:一個 key 只允許一個資料檔定義;加一條斷言(「急救貼片保有戰鬥欄位」)進測試頁防回歸。**多檔合併到同一全域物件時,重複 key 是最陰的 bug 來源。**

### 4.2 讀檔不得重複執行進場效果
- **症狀**:讀檔後關鍵道具變兩個。
- **根因**:讀檔回到節點時重新走了 `Scene.show()`,把該節點的 `onEnter` 效果又執行了一次——但那些效果早已包含在存檔狀態裡。
- **對策**:`show(id, { skipEnter: true })`;所有「恢復現場」路徑(讀檔、匯入、交談返回)一律帶上。原則:**效果只在「第一次抵達」執行,恢復現場只渲染不執行。**

### 4.3 戰鬥結束後的殘留計時器
- **症狀**:偶發 `Cannot read properties of null (reading 'over')`,越到後期 Boss 戰越常見。
- **根因**:敵方 AI、回合推進都用 `setTimeout`;戰鬥結束把 `Combat.cur` 設為 null 後,飛在半空的回呼落地就爆。
- **對策**:每個由計時器喚起的函式,進門第一行 `if (!Combat.cur || Combat.cur.over) return;`。**任何「非同步回呼 + 可被銷毀的狀態」組合都要做這個防禦。**

### 4.4 版本升級會弄壞舊測試
- **症狀**:存檔版本從 v1 升 v2 後,「遷移 v0→v1」的測試紅了。
- **根因**:遷移是鏈式的,v0 存檔現在會一路升到 v2;舊斷言寫死 `version === 1`。
- **對策**:遷移測試一律斷言「升到 `Save.VERSION`(最新)」而非固定數字,並檢查**每一代**補上的欄位都存在。

### 4.5 內容數值 vs 成長系統的脫鉤
- **症狀**:第五章夥伴血量還停在第一章水準,被 Boss 一輪打空。
- **根因**:夥伴 HP 是內容資料裡的靜態常數,玩家卻在升級——寫內容時很自然,跑到後期就崩。
- **對策**:凡是會被「成長曲線」追過的數值,一律改為**公式**(`基礎值 + (等級-1)×4`),資料檔只放基礎值。
- **同場加映**:夥伴的「留守被動」上線很久都只是一段描述文字,從未接上機制,其中一條還把開發里程碑代號「M3 啟用」直接漏給玩家看見。教訓:**UI 上承諾的機制,要嘛做掉,要嘛別寫**;玩家截圖是最好的 QA。

### 4.6 「必勝關卡」必須有保底路徑
- **症狀**:自動化測試(以及低配隊伍的真人玩家)在終章屏障艦戰陷入「戰敗→修船→再戰敗」的無限迴圈。
- **根因**:劇情必經的強制戰鬥,難度假設了玩家有升級船體,基礎船體在數學上打不贏。
- **對策**:每一場**不可繞過**的戰鬥都要有敘事保底——本作給了「呼叫盟軍艦隊集火掩護」(把第四章結盟的成果變成兌現點,順便是好劇情)與「鉚釘識別碼和平通過」。原則:**卡關迴圈是設計 bug,不是玩家太弱**;戰敗路徑要嘛推進劇情、要嘛開新出口。

### 4.7 檢定失敗不擋主線(fail-forward)
所有 d20 檢定的失敗分支都通往「不同代價的路」,而非死路:失敗 = 扣血/驚動守衛/繞遠路,劇情照走。這讓自動化測試可以「無腦點第一個選項」跑通全五章——**如果機器人隨便亂點都不會卡死,真人玩家也不會。**

### 4.8 行動版:桌面版縮小 ≠ 行動版
- **症狀**(玩家實機截圖回報):底部面板要橫向拖拉才找得到系統按鈕;戰鬥選單外溢、跟底部面板疊在一起;劇情區被壓到只剩三行。
- **根因**:RWD 只做了「欄位變一欄」,沒有重新思考行動版的資訊優先序;絕對定位的戰鬥覆蓋層內容超高後溢出容器(沒有 `overflow-y: auto` 就會直接畫在下層面板上)。
- **對策**:
  1. 行動版把最常用的(系統快捷)排最前,次要資訊(任務/所在地)並排壓縮;
  2. 面板做**可收合**,把高度還給主內容;
  3. 一切 `position:absolute; inset:0` 的覆蓋層都補 `overflow-y:auto`;
  4. 結算訊息出現時 `scrollIntoView()`,不然按鈕在畫面外。
- **教訓:行動版問題幾乎都是真機截圖才暴露的——模擬器寬度測不出「拇指找不到按鈕」。**

### 4.9 Web Audio 的三個教訓
1. **持續音 ≠ 音樂**:第一版背景音樂是三個鋸齒波 + LFO 的靜態墊音,玩家的原話是「只是噪音」。要有音樂性,最少要件是:和弦**進行**(會動的和聲)、有節奏錨點、有稀疏旋律。生成式(規格+隨機)比寫死音序省工又不重複。
2. **手勢解鎖**:`AudioContext` 在使用者互動前是 suspended。集中在全域 click 監聽裡 `resume()`,別散落各處。
3. **exponentialRamp 不接受 0**:目標值用 `0.0001`,峰值下限 `Math.max(0.001, peak)`——否則某些參數組合直接拋例外,而且只在特定音效觸發時發生,極難追。

### 4.10 混入敘事的外文與佔位字樣
- **症狀**:玩家截圖圈出劇情裡的「一道 **still** 完好的艙門」;另有一處「first次」。
- **根因**:大量產出中文文本時混入英文思考殘留。
- **對策**:一條 grep 就能當 lint:`grep -rnoE '[一-鿿][a-zA-Z]{2,}|[a-zA-Z]{2,}[一-鿿]' data/ | grep -v "合法縮寫清單"`。專有縮寫(RIVET、EP、DC)做白名單。

### 4.11 UI 自動化測試的三個穩定性技巧
1. **重繪會 detach 按鈕**:戰鬥每回合重繪操作列,`click()` 到一半元素就沒了 → 所有點擊包一層「失敗靜默重試」的 safeClick。
2. **樞紐節點會讓「永遠點第一項」死循環**(整補↔返回互指)→ 走訪計數,同節點造訪 ≥3 次改點**最後一項**(慣例上是「離開/推進」)。
3. **隨機機制的測試**:機率型(製造暴擊、燃料節省)用大樣本+寬區間斷言;能注入的就注入(`Check._rng` 可替換,測骰 1/骰 20 的臨界行為)。浮點也算「隨機」:`10 × 1.15 = 11.499...`,`Math.round` 後是 12 不是 11——斷言前先用 node 算一遍。

### 4.12 交談/彈窗式支線的「返回」問題
從小隊面板進入夥伴對話後要回到**原本的敘事節點**。節點資料寫不了動態返回目標 → 引擎記 `Core._returnNode`,對話結尾用 `{ action: "resume" }`。凡是「暫離主線的插入式內容」都需要這條返回通道,而且返回要走 `skipEnter`(見 4.2)。

## 五、測試策略

### 5.1 三層防線
| 層 | 載體 | 內容 | 執行成本 |
|---|---|---|---|
| 單元斷言 | `test.html`(免建置,瀏覽器開啟即跑) | 檢定數學、DSL 解析、存檔 round-trip、遷移鏈、戰鬥公式、被動加成 | 秒級 |
| 內容 lint | 同上 | 見 5.2 | 秒級 |
| 端對端 | Playwright 腳本 | 創角→五章通關→四結局門→行動版量測,約 60 項檢查 | ~4 分鐘 |

### 5.2 內容 lint:分支敘事的安全網
資料是宣告式的,所以可以整棵掃描:
- 每個 `goto` / `check.success` / `check.fail` / `combat` / `shipCombat` 指向的目標**必須存在**(抓斷鏈)
- 遭遇表引用的敵人/客串/勝敗節點存在
- 星圖航線**雙向對稱**、未鎖定星系必有抵達節點
- 個人任務從起點做圖走訪,**每條分支都能抵達完成節點**
- 商店品項、科技前置、配方材料引用存在

**這套 lint 抓到的斷鏈,比人工試玩抓到的多一個數量級。** 寫分支敘事而不寫內容 lint,等於裸奔。

### 5.3 端對端的核心主張
讓機器人用「最笨策略」(點第一個選項、戰鬥只按攻擊)通關全篇。它同時驗證了:內容無死路(4.6/4.7)、難度下限合理、存讀檔中途不壞。**跑不完的不是測試太笨,是遊戲有洞。**

## 六、開發流程與部署

- **里程碑切分**:M0 引擎骨架(一段示範劇情跑通「選項→檢定→旗標→存讀檔」)→ M1 創角+序章+戰鬥 → M2 星圖+第一章 → M3 系統+第二章 → M4 三四章 → M5 終章+結局。**每個里程碑以「可玩到哪裡」定義完成**,每次收尾跑全套驗證再前進。
- **部署**:GitHub Pages 直接吃 `main` 分支根目錄,零建置;功能分支 → PR → 合併 → 約 30 秒自動上線。上線後用 `curl` 輪詢新檔案回 200 確認部署完成。
- **玩家回饋閉環**:本專案最有價值的三個修正(行動版版面、背景音樂、佔位字樣外洩)全部來自**真機截圖**。上線要早,回饋通道要通。
